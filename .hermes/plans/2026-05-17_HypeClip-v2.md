# 🎬 HypeClip — Zero-Login Clip-to-Shorts Tool

## Goal

Outil web **100% public, sans inscription ni login**. Tu arrives, tu cherches un streameur, tu vois ses clips trending 24h, tu en sélectionnes, ça te génère des Shorts. Point.

**Pas de DB, pas de users, pas de comptes.**

## Ce qu'on réutilise d'Hypesync

| Quoi | Comment |
|------|---------|
| Client-ID Twitch GraphQL | `kimne78kx3ncx6brgo4mv6wki5h1ko` (même approach, requêtes GraphQL non authentifiées) |
| Pipeline de montage FFmpeg | `fast_pipeline.py`, `webcam_extractor_optimized.py`, `clip_processor.py` — le système de crop 9:16, overlay webcam, etc. |
| Clip downloader | `clip_downloader.py` — download des clips via signed URLs |

## Ce qu'on NE fait PAS

- ❌ Pas de login/register
- ❌ Pas de DB (pas de SQLite, pas de users)
- ❌ Pas de sous-titres (pas de Whisper, pas de subtitle overlay)
- ❌ Pas de scheduler
- ❌ Pas d'upload YouTube/TikTok
- ❌ Pas de lien avec Hypesync (projet totalement indépendant)

---

## User Flow

```
Page d'accueil (barre de recherche)
    → Tape un nom de streameur
    → Résultat : grille de clips trending 24h (thumbnail, titre, vues, durée)
    → Sélectionne 1-N clips (checkboxes)
    → Clique "Générer"
    → Options : Webcam ON/OFF (OFF par défaut) | Nom streamer ON/OFF (OFF par défaut)
    → Processing (progress par clip)
    → Résultat : liste des Shorts générés avec preview + download
```

## Step-by-Step Plan

### Phase 1 — Scaffold + Page Recherche

**Step 1.1: Setup Flask**
- `app/__init__.py` — Flask factory minimal (pas de DB, pas de login)
- `requirements.txt` — Flask, requests, opencv-python, ffmpeg-python
- `run.py` — entry point sur port 5002

**Step 1.2: Layout de base**
- `base.html` — Tailwind CDN, theme dark/epuré, zero navbar complexe
- Juste un logo "HypeClip" + barre de recherche centrée

**Step 1.3: Twitch Service**
- `twitch_service.py` — 2 méthodes :
  - `search_streamer(query)` → GraphQL query pour valider le streamer (login, display_name, profile_image)
  - `get_trending_clips(streamer_login, period="LAST_DAY", sort="TRENDING")` → `ClipsCards__User` query
- Retourne : slug, title, view_count, duration, thumbnail_url, created_at

**Step 1.4: Pages**
- `index.html` — Barre de recherche (auto-complete streamer)
- `clips.html` — Grille responsive de clips avec checkboxes, tri par views

---

### Phase 2 — Pipeline Montage (Copy + Adapt)

**Step 2.1: Copier les scripts nécessaires**
- Copier depuis Hypesync `tools/tiktok/` :
  - `clip_downloader.py`
  - `fast_pipeline.py`
  - `webcam_extractor_optimized.py`
  - `clip_processor.py`
  - `combine_videos.py`
  - `fonts/Heavitas.ttf`
- Adapter les imports pour standalone (pas de dépendance Flask/Hypesync)

**Step 2.2: Shorts Service**
- `shorts_service.py` — Pour chaque clip sélectionné :
  1. **Download** clip via `clip_downloader`
  2. **Montage** via `fast_pipeline` adapté :
     - Webcam OFF (défaut) : crop direct 9:16, pas d'extraction webcam
     - Webcam ON : pipeline complet (detection + overlay)
     - Nom streamer OFF (défaut) : pas de text overlay
     - Nom streamer ON : ajouter nom en overlay
  3. **Output** : MP4 9:16 dans `output/{session_id}/`

**Step 2.3: Routes**
- `POST /generate` — reçoit la liste de clips + options (webcam, nom)
- `GET /result/{session_id}` — affiche les Shorts finis
- `GET /download/{session_id}/{filename}` — download du fichier

## Options de Customisation

| Option | Défaut | Description |
|--------|--------|-------------|
| Webcam | **OFF** | Extraction + overlay webcam du streamer |
| Nom du streamer | **OFF** | Text overlay avec le nom du streamer |

Pas d'autres options en V1. Keep it simple.

---

## Risques

1. **Concurrence** — Si 10 users génèrent en même temps, le serveur CPU sature. **Mitigation :** Queue simple (1 job à la fois) en V1.
2. **Storage** — Les Shorts prennent de la place. **Mitigation :** Auto-cleanup après 1h.
3. **Twitch rate limit** — Le GraphQL non-auth est limité. **Mitigation :** Cache les résultats de recherche 5 min.
